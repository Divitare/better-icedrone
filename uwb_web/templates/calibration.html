Help & Tuning Guide

Hover any underlined label for a quick explanation. Expand sections below for detailed guidance.

Kalman Filter (EKF)

The Extended Kalman Filter tracks position and velocity over time. It works like image stabilisation — using temporal coherence to smooth out noise.

  • When to enable: Always recommended for a moving tag. You can disable it for static/diagnostic tests.
  • Process Noise controls responsiveness:
    • 0.01–0.05 — very smooth; best for slow or stationary tags
    • 0.1 default — good balance for walking speed
    • 0.5–1.0 — follow fast motion / quick direction changes
  • Range Variance tells the filter how noisy the raw UWB ranges are:
    • 0.01 — very clean environment, short range, calibrated anchors
    • 0.1 default — typical indoor UWB
    • 0.3–0.5 — long range or heavy multipath
  • After calibration, per-anchor variances are computed automatically from the calibration std error — the default range variance is only used for uncalibrated anchors.
  • Reset EKF when you physically move the tag to a new location, otherwise the filter will try to smoothly track to the new position.
NLOS Rejection

In indoor environments, UWB signals can bounce off walls, creating reflected paths (NLOS) that make the anchor report a longer range than the true distance.

  • How it works: After an initial position solve, each anchor's expected range is compared to the measured range. The worst outlier is dropped and the position is re-solved. This repeats until all residuals are below the threshold (or only 3 anchors remain).
  • NLOS Threshold — the residual cutoff in metres:
    • 0.3 — aggressive; use with many anchors (≥ 6)
    • 0.5 default — balanced
    • 0.8–1.0 — permissive; only reject very bad reflections
  • Minimum anchors: Rejection requires ≥ 4 anchors so that at least 3 remain after a drop. If you only have 3, disable this.
Calibration Workflow

Calibration measures 'ground truth' positions with the motion controller and compares them to UWB ranges to compute per-anchor bias and scale corrections.

  • Grid setup: Aim for at least 9 points (3 × 3) spread across the working area. More points = more robust fit.
  • Dwell time: How long to average ranges at each point. 2–5 s is typical; longer improves noise averaging but takes more time.
  • Apply Corrections: Saves per-anchor bias/scale to the system, immediately correcting live ranges.
  • After applying, the Position Engine automatically loads the new per-anchor variances — anchors with lower calibration error get higher weight.
Trajectory Smoother ("Deblur")

The RTS smoother is the equivalent of deblurring a photograph — it reprocesses an entire recorded trajectory using both past and future data to produce the sharpest possible path.

  • When to use: After a calibration run or any recorded session. It cannot improve real-time tracking (which only sees the past).
  • Process noise & measurement noise parameters work the same way as in the Kalman Filter section.
  • The smoother is most effective on trajectories with consistent motion — if the tag was stationary, there's nothing to 'deblur'.
Quick-Start Recommendations
  • Enable both EKF and NLOS rejection.
  • Run a 3 × 3 calibration grid covering most of your working area.
  • Apply corrections — this also loads per-anchor weights into the engine.
  • Use Estimate Transform to auto-compute the origin offset and rotation — much more accurate than measuring by hand.
  • Use Refine Anchors to optimise anchor positions using your calibration data.
  • Start with the defaults (process noise = 0.1, NLOS threshold = 0.5, range var = 0.1).
  • If the position jumps too much: lower process noise (e.g. 0.05).
  • If the position lags behind motion: raise process noise (e.g. 0.3).
  • If good anchors get dropped: raise NLOS threshold (e.g. 0.8).
  • If a bad anchor isn't removed: lower NLOS threshold (e.g. 0.3).
Auto-Origin (Coordinate Alignment)

Instead of measuring the offset between the motion controller and UWB room origin by hand (which introduces error), the system can compute it automatically from calibration data.

  • How it works: The motion controller moves the tag to a grid of points. The relative positions between grid points are very precise (sub-mm). The UWB system also provides a (noisy) position estimate at each point. An SVD-based Procrustes algorithm finds the optimal rotation + translation + scale that aligns the two coordinate frames.
  • What it replaces: The manual Origin X/Y/Z offset. After running this, the origin offset is computed to sub-centimetre accuracy rather than relying on a tape measure.
  • When to use: After completing a calibration run. You can set Origin to (0, 0, 0) and let auto-origin handle the alignment.
  • Scale factor: Should be very close to 0.001 since input is mm and output is metres. A large deviation may indicate a setup issue.
Anchor Position Refinement

Anchor positions are typically measured by hand with a tape measure — this can easily be off by several centimetres. Since the calibration grid provides precise relative tag positions and per-anchor range measurements, we can optimise the anchor coordinates to better explain all the observed ranges.

  • How it works: A Gauss-Newton optimiser adjusts each anchor's (x, y) position to minimise the total range residual across all calibration points. The tag positions are held fixed (they come from the precise motion controller + rigid transform).
  • Typical improvement: 2–10 cm shift per anchor, depending on how precisely you measured them initially.
  • Workflow:
    1. Run a calibration grid (3×3 or larger)
    2. Click Estimate Transform to auto-compute the origin
    3. Click Refine Anchors to optimise anchor positions
    4. Review the deltas (ΔX, ΔY) — they should be reasonable (a few cm)
    5. Click Apply Refined Positions to save to the database
    6. Re-run calibration or Apply Corrections to recompute range corrections with the improved anchors
  • Caution: If the grid is very small relative to the room, the refinement may not fully constrain distant anchors. Use the largest practical grid.
